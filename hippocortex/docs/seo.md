## Indexing tips

1. Keep the README title and opening paragraph stable; search engines reward consistent semantics.
2. Use architecture terms repeatedly but naturally (episodic memory, semantic memory, long-term memory).
3. Keep docs paths stable (`docs/architecture.md`, `docs/architecture.mmd`).
4. Include runnable examples to reduce bounce and improve engagement signals.
5. Add descriptive alt text for images and keep diagram filenames stable.

## PR/release checklist

- [ ] README updated for user-facing changes
- [ ] `CHANGELOG.md` entry added
- [ ] Quickstart still runs locally
- [ ] Architecture diagram regenerated if architecture changed
- [ ] Governance docs still valid (`CONTRIBUTING`, `SECURITY`, `CODE_OF_CONDUCT`)
- [ ] Version/tag strategy reviewed
- [ ] GitHub topics reviewed for current positioning
